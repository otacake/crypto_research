# style_guide.md

## 目的
このガイドは、原稿を「正しいが読みにくい文章」から「正しく読みやすい文章」に変えるための運用基準です。

## 想定読者
- 理系の素養はある
- IT基礎語に不慣れ
- 定義と順序が適切なら理解できる

## 基本原則
- 読者の脳内で場面が再現できる順に書く。
- 1段落1メッセージで、段落間の接続を明示する。
- 事実主張は出典を付け、未確証は断定しない。

## 段落の役割
各段落は次のいずれか1つに限定する:
- 状況提示
- 問い提示
- 反例提示
- 要件提示
- 仕組み説明
- まとめ

## 可読性ルール（実務）
- 1段落内の新規用語は最大2個。
- 2文目までに段落の主張を出す。
- 破綻説明は最低2つの具体例を入れる。
- 破綻から要件へは橋渡し文を1〜3文で入れる。
- 同趣旨の言い換え反復をしない。

## 禁止パターン
- 読点ごとの改行
- 定義前用語の連発
- 指示語だけで参照先が不明な文
- 章内で同じ主張の重複展開
- 根拠なし断定

## 出典運用
- 事実主張: `[一次:S-xxx]` または `[二次:R-xxx]`
- 未確証: まず断定回避で書き換える
- それでも必要な場合のみ `[要検証:RB-xxx]` を使う（目安: 1節0〜1件）

## 執筆前チェック
- この節で読者に理解させる主張は1つに絞れているか
- 必要な用語定義は先に置けているか
- 本文が `ops/outline.md` と `ops/claims.md` に整合しているか

## 執筆後チェック
- 段落ごとに役割が明確か
- 重複段落がないか
- 用語の初出定義漏れがないか
- 章文字数が `ops/budget.md` の範囲内か
