# AGENTS.md (repo root)

## 目的
本リポジトリは、日本語の原稿を作る。  
狙いは「ブロックチェーン=暗号資産の発行技術」ではなく、「中央裁定者がいない環境で、取引の順序を共有する技術」として理解させること。

## 成果物と編集ルール
- 編集の本体: `docs/manuscript/chapters/*.md`
- 生成物: `docs/manuscript/master.md` は章を結合し、参考文献を `docs/references/sources.yaml` から自動付与して生成する。手編集しない。
- `master6.md` は現時点では触らない。

## 設計参照（執筆前に確認）
- `ops/blueprint.md`（全体方針・中心命題・用語導入順）
- `ops/outline.md`（章→節のメインメッセージ）
- `ops/claims.md`（主張IDと必要根拠タグ）
- `ops/budget.md`（章/節の文字数予算）
- `docs/style_guide.md`（文章品質の運用基準）
- `docs/glossary.yml`（用語定義）
- `docs/references/sources.yaml`（一次資料候補）

## 最優先品質方針
- 読者の頭の中に「場面」と「因果」を作れない文は不採用。
- 1段落1メッセージ。1段落内で新規用語を詰め込みすぎない。
- 同一節内で同じ主張を言い換え反復しない。
- 段落は「前段との接続語」または「問い」で始め、ジャンプを防ぐ。

## 出典運用（統一必須）
- 引用と出典は、標準Markdown記法のみを使う。
- 事実主張には文末でMarkdown脚注（例: `[^S-xxx]`）または通常リンク（`[資料名](URL)`）を付ける。
- 直接引用はMarkdownの引用記法（`>`）を使い、直後に出典を付ける。
- 未確証の主張はまず断定を避ける。それでも必要な場合のみ「要検証」と明記し、脚注（例: `[^RB-xxx]`）で管理する。
- `S-xxx` は `docs/references/sources.yaml`、`RB-xxx` は `ops/research_backlog.md` と一致させる。
- カスタムタグ（`[一次:...]` / `[二次:...]` / `[要検証:...]`）は使用禁止。

## 公開リポジトリ表記ルール
- 環境依存の絶対パスを書かない。
- パスは原則リポジトリ相対で書く。
- ローカルユーザー名やローカル構成を公開物へ残さない。

## 文章品質ルール（必須）
- 読点ごとに改行しない。改行は段落境界のみ。
- 曖昧語（例: これ/それ/置く/進める）を連発しない。
- 「破綻」は抽象で終えず、観測者・時点・結果が異なる具体例を示す。
- 「破綻→要件」の橋渡しを1〜3文で明示する。
- 用語は初出で定義する。略語は英語フルスペル+日本語説明+略語を併記する。

## 役割分離
- 執筆: `roles/writer/AGENTS.md`
- 読者レビュー: `roles/reader/AGENTS.md`
- 反映編集: `roles/editor/AGENTS.md`

Codexは repo root から現在ディレクトリまでの AGENTS を連結して読み、後勝ちで適用する。
