# スコープ定義

## 目的
本原稿の目的は、ブロックチェーンを「暗号資産の発行技術」としてではなく、「中央裁定者がいない環境で取引順序を共有する技術」として理解できるようにすること。

## 対象読者
- 一般的な理系の素養（論理展開、簡単な数式読解、因果関係の把握）がある人
- 一般的なIT用語（サーバー、ネットワーク、データベース、プロトコル）には慣れていない人
- 分散システムと暗号の初学者
- 読み手としての認知能力は十分にあり、用語定義と説明順序が適切なら内容を正確に理解できる人

## 到達点
- なぜ「順序共有」が必要なのかを、自分の言葉で説明できる
- 中央裁定者あり/なしで何が変わるかを比較できる
- Bitcoinを中心に、取引・ブロック・PoW・合意収束・インセンティブの関係を説明できる
- ブロックチェーンに関する一般的な主張を、前提と根拠の有無で評価できる

## 扱うこと / 扱わないこと
| 区分 | 内容 |
|---|---|
| 扱う | 順序共有問題の定式化、P2P伝播遅延、トランザクションモデル（UTXO）、ブロック構造、Proof of Work、難易度調整、フォークと最終性、攻撃コストの考え方 |
| 扱う | 歴史的な設計判断と、その判断が満たす要件の対応関係 |
| 扱わない | 投資助言、価格予測、銘柄推奨、税務・法務の実務判断 |
| 扱わない | 特定チェーンの最新ニュース追跡、DeFi運用ノウハウ、短期売買テクニック |
| 扱わない | 実装ハンズオン（フルノード実装、マイナー実装）や形式検証の厳密証明 |

## 技術的到達の上限
- 到達する上限:
  - 主要概念を数式なし/最小限の式で追える
  - プロトコル仕様や一次資料を読んで、設計意図を要約できる
  - 基本的な攻撃シナリオ（例: 二重支払い、再編成）の成立条件を説明できる
- 到達しない上限:
  - 暗号理論の証明（安全性証明、還元証明）を厳密に再現すること
  - クライアント実装をゼロから構築すること
  - 研究最前線（高度な合意アルゴリズムやZK詳細）の網羅
