# 文章スタイル規則（運用最小版）

作成日: 2026-02-15  
対象: `docs/manuscript/chapters/*.md`

## 目的
- 読者が途中で離脱しない導線を維持する。
- 断定過多・抽象過多・主語不在を防ぐ。
- 文章の質感（テンポ、温度、接地性）を再現可能にする。
- 参照コーパス（唐鎌・池田・服部）を下回らない説明密度とトンマナを固定する。
- テンプレ進行語の多用を抑え、自然な読み味を維持する。

## 優先順位
1. 読み味（自然さ）ゲート（Q-01〜Q-06, R-39〜R-47）
2. 上位化固定ルール（U-01〜U-11）
3. Core（C-01〜C-06）
4. 追加学習ルール（R-33〜R-38）

## 運用方針
- 本書は最小運用版。上記以外の旧ルール（R-01〜R-32）は運用対象から外す。
- 旧ルールの参照が必要な場合は Git 履歴と `ops/style_corpus.md` を参照する。

## 0) 絶対禁止（本文）
- 本文で `主張は` `実務判断として` `先に固定` を使わない。
- 見出し、レビュー文書、ルール文書での使用はこの禁止対象外。

## 1) Core（必須）
| Core ID | ルール | 制御条件 |
|---|---|---|
| C-01 | 冒頭2文で「何の話か」「なぜ今読むか」を示す | 2文固定。3文目から本文へ入る |
| C-02 | 用語は初出で定義し、その段落で1回使う | 1段落の新規用語は最大2語 |
| C-03 | 競合や選択を扱う節では、新しい仕組みの前に破綻例を置く | 適用時は破綻例2件、各3文以内 |
| C-04 | 断定には必ず境界条件を添える | 条件句は1主張につき1つまで |
| C-05 | 数値を使う節では「比較軸・単位・期間」を同時に書く | 数値提示ごとに3点を最小記載 |
| C-06 | 1段落1判断に固定し、同趣旨反復を禁止する | 同趣旨は連続2文まで |

### 5.4 NGパターン
1. 情報を増やすこと自体を目的化し、必須情報の比率が下がる。  
2. 全論点を同じ熱量で扱い、重要度が見えなくなる。  
3. 個性を出すために比喩・感想を増やし、論理が薄くなる。  
4. 主張の確実度が低いのに、文体だけ強断定へ寄せる。  
5. 章内で語り手の立場が頻繁に切り替わる。

### 6.1 冒頭多様性の機械チェック（必須）
- 判定基準: 同一節内で、連続する段落の「第1文先頭10文字（空白除去）」が一致したらFAIL。
- 実行コマンド（PowerShell）:

```powershell
@'
import re, sys, pathlib
p = pathlib.Path(sys.argv[1])
t = p.read_text(encoding='utf-8')
parts = re.split(r'(?m)^##\\s+', t)
failed = 0
for sec in parts[1:]:
    lines = sec.splitlines()
    title = lines[0].strip()
    body = "\n".join(lines[1:])
    paras = [x.strip() for x in re.split(r'\n\s*\n', body) if x.strip() and not x.strip().startswith(('#','-','```'))]
    prev = None
    prev_i = None
    for i, pa in enumerate(paras, 1):
        first = re.split(r'[。！？!?]', pa, 1)[0]
        key = re.sub(r'\s+', '', first)[:10]
        if prev is not None and key == prev:
            print(f"FAIL {title} para{prev_i}-{i}: {key}")
            failed += 1
        prev = key
        prev_i = i
if failed == 0:
    print("PASS opening-diversity")
'@ | python - docs/manuscript/chapters/<chapter-file>.md
```

## 7) 参照記事超えの上位化固定ルール（必須）
### 7.1 内容上位化
| Rule ID | ルール | 制御条件 |
|---|---|---|
| U-01 | 各節は `主張→根拠→反例/限界→境界条件→実務判断` の順で構成する | 5要素の欠落を禁止 |
| U-02 | 事実主張の出典付与率を100%にする | 事実文には脚注またはリンクを必須化 |
| U-03 | 成立条件と非成立条件を対で記載する | 条件記述が片側のみの節をFAILとする |

### 7.2 トンマナ上位化
| Rule ID | ルール | 制御条件 |
|---|---|---|
| U-04 | 章ごとに語り手の立場を固定する | 1章内で主軸視点を変更しない |
| U-05 | 抽象語は同段落で参照先を明示する | 抽象語の放置を禁止 |
| U-06 | 段落の役割を `導入/展開/収束` で持たせる | 役割不明段落を削除候補にする |
| U-07 | 命令調より観測・判断で閉じる | 命令形終止は1節1回以下 |

### 7.3 運用上位化（PDCAゲート）
| Rule ID | ルール | 制御条件 |
|---|---|---|
| U-08 | Readerは内容スコアとトンマナスコアを分離評価する | 5点満点で両方4点以上をPASS |
| U-09 | どちらかがFAILなら再Doへ戻す | Act完了判定を禁止 |
| U-10 | 各章で「参照記事より弱い箇所」を最低1件特定する | 章ごとに改善ログ項目を1件必須化 |
| U-11 | 同一節で段落冒頭パターンの連続重複を禁止する | 6.1 の機械チェックで FAIL=0 を必須化 |
